# 7 практика

Я решил слегка поменять формат практики. В этот раз я дам вам готовое (но некорректное) решение
и вам нужно будет его исправить. Ошибки есть как в плане логики, так и в плане структуры проекта.

Я хочу чтобы вы форкнули этот репозиторий, внесли правки и сделали пулл-реквест с вашими исправлениями.

## ТЗ
Нужно сделать программу, которая считывает от пользователя от 1 до бесконечности URL до файлов.
В этих файлах на каждой строчке написаны фразы, порядок которых не важен.
Нужно параллельно считать эти файлы из интернета и слить в один локальный файл, путь до
которого также считать у пользователя. В конце вывести суммарное количество строк в итоговом файле.

### Условия
1. Если при чтении файла происходит ошибка, то этот файл игнорируется
2. Нужно валидировать URL, которые вводит пользователь
3. Ни в один момент нельзя целиком выгружать файлы в память, вся работа идёт в режиме потока
4. Если пользователь нажимает стандартную комбинацию для выхода из консольных программ (CTRL+C),
то программа должна удалить файл, в который начала записывать
5. Если пользователь ввёл уже существующий файл, то нужно спросить хочет ли он его перезаписать
6. Файлы должны считываться и записываться параллельно, а не последовательно 
(обязательно `Parallel` или `Task.WhenAll`)
